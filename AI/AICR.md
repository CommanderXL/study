## 工程与模型

先回顾下 AICR 的几次架构升级：

### Prompt workflow

最初 AICR 的演进思路是将24～25年泛前端的事故总计300+，逐条通过 AI 分析一遍，在这个过程中引导 AI 产出识别问题的通用规则，这部分沉淀下来的通用规会被注入到 AICR 的上下文当中来辅助 LLM 进行问题的识别。

* 设计

在一个上下文当中，纯使用 prompt 驱动整个 AICR 的 workflow。

Diff + (CR Prompt + CR Rules) -> LLM 

* 问题

1. 模型推力能力不强，模式识别严重。举个例子... 噪音问题比较严重，错误兜底类的问题占比 50%+；
2. 模型 Context Window 不高，128K，当上下文增加后， LLM 幻觉严重，指令遵循度不高；
